# 闯关小勇士 — 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品定位
一款面向家庭内部使用的**儿童习惯养成 + 学科启蒙 + 妈妈私密工具** App。孩子通过扮演"小勇士"角色，每天完成习惯任务积累星星，星星可兑换家长配置的真实奖励；每天获得固定数量的小心心，用于解锁学习内容；妈妈可在独立的私密工具页记录经期、症状、提醒和周期趋势，帮助家庭照护者更好地安排自己的生活节奏。

产品需要同时照顾两类完全不同的使用场景：
- **儿童场景**：视觉驱动、操作简单、即时反馈、避免误触成人数据。
- **成人私密场景**：记录高效、数据敏感、默认隐藏、强调隐私和可控同步。

### 1.2 目标用户
- **核心用户**：3-6 岁儿童（操作需极其简单，视觉驱动）
- **配置者**：家长（设置任务、奖励、使用时长限制）
- **妈妈工具用户**：女性家长 / 妈妈（记录经期、症状、提醒和个人周期趋势）
- **账号主体**：家长手机号账号（孩子不直接注册账号，只作为家长账号下的孩子档案）

### 1.3 核心价值
| 角色 | 价值 |
|------|------|
| 孩子 | 好玩的游戏 + 看得见的成就感 + 真实的奖励 |
| 家长 | 孩子主动做习惯 + 在玩中学 + 可控的屏幕时间 |
| 妈妈 | 快速记录经期和身体状态 + 提前准备 + 私密数据可控 |

### 1.4 产品 Slogan
**"每天闯一关，习惯自然成！"**

### 2.1 登录与注册入口

首次打开 App 或登录态失效时，先进入登录/注册页面；登录成功后进入四 Tab 主界面。

#### 2.1.1 登录方式
- 使用 CloudBase 身份认证能力。
- MVP 账号使用**手机号格式账号 + 密码**方式维护，手机号作为唯一账号标识。
- 手机号属于家长账号，孩子不直接持有登录账号。
- MVP 暂不接入短信验证码 SDK，不验证手机号实际归属；技术实现按 CloudBase 用户名/密码登录能力处理，用户名使用 11 位手机号格式。
- 该方案仅用于 MVP 内测。正式对外发布前必须接入短信验证码或其他可信手机号归属验证能力。
- 登录成功后，服务端返回 AccessToken / RefreshToken，客户端安全保存登录态。

#### 2.1.4 登录注册规则
- 手机号格式：中国大陆手机号，前端输入 11 位，仅作为账号格式和唯一标识。
- MVP 不发送验证码，也不声明已验证手机号归属。
- 密码长度：8-20 位，不能以特殊字符开头，并需包含小写字母、大写字母、数字、特殊字符中的至少三类。
- 同一手机号格式账号只能注册一次；重复注册时提示直接登录。
- 注册成功后默认创建 1 个孩子档案；若缺少孩子昵称，则进入孩子档案初始化页。
- MVP 暂不提供手机号验证码自助找回密码；“登录遇到问题”页面提示联系内测管理员处理。
- 正式上线前，密码找回必须与短信验证码等可信身份验证能力一并补齐。
- 连续输错密码达到阈值后短时间限制尝试，避免撞库。
- 退出登录后清除本地 AccessToken，但保留必要的本地缓存；下次登录同账号后重新同步。

### 2.2 底部导航栏设计
- 四个 Tab 等宽分布，图标在上 + 文字在下
- 当前选中 Tab 图标放大 + 彩色高亮，未选中为灰色
- 图标尺寸：选中 28dp，未选中 24dp
- 文字字号：12sp
- Tab 文字：任务 | 妈妈 | 学习 | 我的

### 2.3 四 Tab 设计合理性评估

当前四 Tab 结构整体合理，适合 MVP 阶段继续沿用：

| Tab | 核心角色 | 是否合理 | 调整建议 |
|-----|---------|---------|---------|
| 任务 | 每日入口 + 习惯打卡 | 合理 | 建议在 UI 文案上可考虑命名为"今日"，让孩子更容易理解这是每天要做的事 |
| 妈妈 | 女性家长私密工具 | 需要隐私隔离 | 建议承载经期记录、症状记录、提醒和周期趋势；默认隐藏敏感内容，进入后可要求私密解锁 |
| 学习 | 数学 + 英语学习合集 | 合理 | 将原数学 Tab 和英语 Tab 合并到学习 Tab 下，用两个大入口承载后续内容 |
| 我的 | 孩子成就 + 家长设置 | 基本合理 | 孩子默认只看到成就信息；连续点击头部区域 5 次后进入家长区 |

**推荐原则：**
- 不增加第 5 个 Tab，避免 3-6 岁孩子选择成本变高。
- "任务"是孩子每天进入 App 的默认首页；MVP 只承载习惯任务，不加入数学或英语推荐关卡。
- "妈妈"是成人私密入口，不应在任务页、孩子成长页或通知文案中暴露具体经期、症状、备孕等敏感内容。
- "学习"承接原数学和英语两个方向，入口上应先呈现"数学小岛"和"英语森林"两张大卡，而不是把数学和英语继续拆成两个底部 Tab。
- "我的"首页先呈现孩子喜欢的头像、勋章和成长数据；任务管理、奖励配置、数据看板、重置数据等功能仅在家长区显示。
- 数学和英语不要做成纯课程列表，要继续保持"地图/树/小镇"的世界观，孩子才有持续探索感。

### 2.4 CloudBase 云端边界

当前版本计划引入 CloudBase，将用户身份、业务数据库和孩子成长记录放到云端维护。

**云端能力：**
- CloudBase Auth：MVP 使用用户名/密码能力，用户名采用 11 位手机号格式。
- CloudBase MySQL：维护孩子档案、习惯、打卡、学习进度、勋章、奖励等业务表；妈妈健康数据属于高度敏感数据，MVP 默认先本地保存，若启用云同步必须单独授权、单独表隔离并提供删除能力。
- CloudBase 云函数 / HTTP API：承接移动客户端无法直接安全完成的注册、登录、数据读写、统计汇总等操作。
- CloudBase 云存储：后续可存头像、装扮、音频、题目素材等资源。

**本地能力：**
- Drift / SQLite 作为本地缓存层，用于提升打开速度和弱网体验。
- MVP 安装包只内置应用基础图片、图标和音效，不内置数学题、英语描边、词汇发音或绘本资源。
- 数学、英语和绘本内容包的版本检查、下载、校验与离线缓存放到后续阶段。
- 妈妈工具的经期记录、症状记录和提醒设置优先保存在本地数据库；若用户未显式开启云同步，不上传到 CloudBase。
- 网络不可用时允许浏览最近同步的数据；涉及写入的操作先进入本地待同步队列。

**仍不做的能力：**
- 不做排行榜、社交分享、陌生人互动。
- 不接入在线视频或外部跳转链接。
- 不做第三方广告、内购和跳转浏览器。
- MVP 不提供可玩的数学关卡、英语课程或分级绘本，只在学习 Tab 保留数学小岛和英语森林占位入口。
- 妈妈工具不提供疾病诊断、避孕结论、妊娠判断或医疗建议；所有预测只作为生活提醒，不作为医学依据。
- 不将妈妈健康数据用于广告、推荐、训练模型或儿童成长数据分析。

**可预留但暂不实现：**
- 多设备数据同步冲突处理。
- 家长手动导入新的内容资源包。
- 语音识别类能力先作为 V2，可先用"家长确认孩子是否愿意开口"替代。

### 2.5 MVP 产品规则冻结

以下规则作为 MVP 开发基线：

| 主题 | MVP 决策 |
|------|---------|
| 星星 | 完成任务获得；同时记录累计获得数和当前可用余额；兑换只扣当前余额，当日撤销或纠错可回滚未最终结算的星星 |
| 奖励币 | 不新增独立奖励币，星星直接承担奖励兑换货币的作用 |
| 小心心 | 每天 0 点自动补满至 10 颗，不累加、不结转；MVP 仅展示规则，学习模块正式上线后才产生实际消耗 |
| 勋章 | 只用于任务成就展示，不消费、不兑换 |
| 打卡撤销 | 只允许修改当天记录；撤销完成时同步撤回该任务产生的当前余额和累计获得数，次日后历史记录只读 |
| 全完成奖励 | 当天全部任务完成时额外获得 2 颗星星；若当天撤销后不再全部完成，则同步撤回奖励 |
| 连续奖励 | 当前日与此前连续 6 天均全部完成时额外获得 5 颗星星；由服务端幂等结算，不重复发放 |
| 登录注册 | 手机号格式账号 + 密码注册登录；MVP 不发送短信验证码，仅限内测 |
| 家长入口 | 在“我的”页面连续点击头部区域 5 次进入家长区；该方式只隔离儿童误触，不等同于安全身份认证 |
| 妈妈工具 | Tab 2 替换为妈妈工具；MVP 先做私密入口、经期日历、快速记录、基础预测和提醒；默认本地保存，云同步需单独授权 |
| 学习内容 | 原数学和英语合并到 Tab 3“学习”；MVP 仅显示数学小岛、英语森林两个占位入口，不实现完整关卡、课程、题库、音频和绘本 |

### 3.2 顶部状态栏

顶部状态栏横向排列三个指标，等宽分布。

#### 3.2.1 星星 ⭐
- **含义**：孩子完成习惯任务后获得的成长里程碑，可用于兑换家长配置的真实奖励。
- **获取规则**：
  - 每完成 1 个任务 = +1 ⭐
  - 当天全部任务完成 = 额外 +2 ⭐
  - 连续 7 天全部完成 = 额外 +5 ⭐
- **数量口径**：
  - `累计获得星星`：兑换奖励时不扣减，用于展示长期成长里程碑；当日撤销打卡或服务端纠错时同步回滚尚未最终结算的星星。
  - `当前可用星星`：获得时增加，兑换奖励成功时扣减。
- 星星不用于解锁学习内容。
- 点击星星可查看累计获得数、当前可用余额和星星获取/兑换记录。

#### 3.2.2 勋章 🏅
- **含义**：任务系统中的成长成就展示，用于记录连续完成、累计完成和习惯稳定度。
- **显示格式**：`已获得数 / 总数`（如 `5/20`）
- 点击跳转至勋章墙（"我的"页面内的勋章区域）
- 获得新勋章时，勋章图标抖动 + 红点角标提示
- 勋章不可消费，也不作为真实奖励的兑换货币。

#### 3.2.3 小心心 ❤️
- **含义**：每日能量心，用于解锁学习关卡
- **每日固定 10 颗**，每天 0 点自动补充至 10 颗，不累加、不结转。
- **显示格式**：`今日剩余 / 10`（如 `7/10`）
- **消耗规则**：
  - 在“学习”Tab 中解锁 1 个数学普通关 = 消耗 1 ❤️
  - 在“学习”Tab 中解锁 1 批英语字母 = 消耗 2 ❤️
  - Boss 关不消耗小心心
  - 同一关卡解锁后永久有效，不重复扣
- **每日补充**：按服务端记录的家庭时区判断日期，每天 0 点恢复至 10 颗，避免通过修改设备时间重复补充。
- **用完后**：关卡入口显示 🔒 + 灰色态，点击提示"今天的小心心用完啦，明天再来吧！"

### 3.3 任务列表

#### 3.3.1 任务行布局
```
┌──────────────────────────────────────────┐
│  [图标]  [任务名称(≤7字)]        [✓] [✗]  │
└──────────────────────────────────────────┘
```
- 每行固定高度 56dp
- 左侧：任务图标（40x40dp，圆角方形底色 + emoji/icon）
- 中间：任务名称，最多 **7 个汉字**（约 14 个英文字符宽度）
- 右侧：✓ 和 ✗ 两个操作按钮

#### 3.3.2 ✓（完成）按钮交互
- **初始态**：灰色空心圆圈 ○（未打钩）
- **点击 ✓**：
  - ○ 变为绿色实心 ✓ 圆
  - 按钮放大回弹动画（200ms）
  - 星星从该行飞入顶部 ⭐ 计数 +1
  - 清脆"叮咚"音效
- **已打钩后再次点击 ✓**：撤销完成，✓ 恢复为 ○，星星 -1（防误触）
- **逻辑互斥**：✓ 和 ✗ 不可同时选中

#### 3.3.3 ✗（跳过）按钮交互
- **初始态**：灰色空心圆圈 ○
- **点击 ✗**：
  - ○ 变为橙色 ✗ 圆
  - 按钮缩小再恢复动画
  - 柔和"咚"音效（表示跳过，不奖励不惩罚）
- **已打叉后再次点击 ✗**：撤销跳过，恢复为 ○
- 跳过不影响星星计数，仅为记录今天这项没做

#### 3.3.4 任务状态总结
| 状态 | ✓ 按钮 | ✗ 按钮 | 视觉效果 |
|------|--------|--------|---------|
| 未操作 | ○ 灰圈 | ○ 灰圈 | 默认白底 |
| 已完成 | ● 绿底✓ | ○ 灰圈 | 整行浅绿底色 |
| 已跳过 | ○ 灰圈 | ● 橙底✗ | 整行浅灰底色 |

#### 3.3.5 全部完成反馈
- 当天全部任务标记为 ✓ → 全屏彩带飘落 2 秒 + "太棒了！全部完成！"弹窗 + ⭐+2 飞入动画

### 3.5 幼升小习惯库与指标

幼升小阶段的习惯目标不是"做很多事"，而是让孩子逐步稳定形成：生活自理、规则意识、专注学习、表达沟通、运动健康和入学准备。

#### 3.5.1 习惯分类

| 分类 | 目标 | 适合任务示例 |
|------|------|-------------|
| 生活自理 | 能照顾自己的基本生活 | 自己刷牙、自己洗脸、自己穿衣、整理水杯、睡前如厕 |
| 作息健康 | 稳定上学节奏 | 早睡、按时起床、吃早餐、喝水、户外活动 |
| 学习准备 | 建立低压力学习节奏 | 读绘本、听故事、握笔练习、坐姿练习、专注 10 分钟 |
| 书包与物品 | 为小学物品管理做准备 | 整理书包、检查文具、收玩具、物归原位 |

## 四、Tab 2：妈妈工具（经期记录 MVP）

妈妈工具是面向女性家长的私密工具页，首个版本重点做**经期记录 + 身体状态记录 + 周期预测 + 温和提醒**。它不参与孩子任务、星星、小心心、勋章和学习激励，也不在孩子可见页面展示任何敏感内容。

### 4.1 需求调研摘要

参考苹果健康的经期跟踪能力和主流经期记录产品，女性用户的通用需求可以归纳为五类：

| 需求点 | 典型功能 | MVP 处理 |
|--------|----------|----------|
| 记录经期 | 开始日期、结束日期、流量、点滴出血 | 必做 |
| 记录身体状态 | 痛经、头痛、乳房胀痛、痘痘、疲劳、睡眠、消化、情绪等 | 必做基础项，后续可扩展 |
| 预测和提醒 | 下次经期预测、经期将至提醒、记录提醒、周期偏差提醒 | MVP 做基础预测和本地提醒 |
| 备孕/排卵参考 | 易孕期、排卵日、基础体温、宫颈黏液、同房记录 | V2；MVP 仅预留，不默认展示 |
| 隐私和导出 | 私密锁、通知脱敏、删除数据、导出给医生 | 私密锁和删除必做；导出 V2 |

苹果健康的经期跟踪提供经期/症状/点滴出血记录、预测、通知、易孕窗口、基础体温、周期因素、历史 PDF 导出和周期偏差提醒；这些能力适合作为功能基线，但本产品 MVP 不应照搬全部医学/备孕能力，先服务“日常记录和提前准备”。

### 4.3 首次设置流程

首次进入妈妈工具时，先进入轻量初始化页：

1. 选择是否开启妈妈工具；未开启时 Tab 只显示空状态和开启按钮。
2. 录入最近一次经期开始日期。
3. 录入通常经期持续天数，默认 5 天，可调整为 2-10 天。
4. 录入通常周期长度，默认 28 天，可调整为 21-45 天。
5. 选择提醒偏好：经期前提醒、每天记录提醒、经期结束确认提醒。
6. 选择数据保存方式：默认仅本地保存；云同步需要单独开启和确认。

### 4.4 快速记录

#### 4.4.1 经期记录
- 点击“来了”后，将当天标记为经期开始日。
- 点击“结束”后，将当天标记为经期结束日。
- 支持回填最近 90 天的记录，避免用户首次使用时无法补历史。
- 支持编辑单日状态：经期中、点滴出血、非经期。
- 同一日期只能有一条主状态；重复记录时覆盖当天状态，不生成重复记录。

#### 4.4.2 流量记录
流量用于帮助用户回顾身体状态，不做医学判断。

| 选项 | 含义 |
|------|------|
| 无 | 当日无经血 |
| 点滴 | 少量点滴出血 |
| 少 | 少量 |
| 中 | 中等 |
| 多 | 较多 |

#### 4.4.3 症状与状态
MVP 建议先提供常见、低负担的记录项：

| 分类 | 记录项 |
|------|--------|
| 疼痛 | 痛经、头痛、腰酸、乳房胀痛 |
| 情绪 | 烦躁、低落、焦虑、平静、开心 |
| 身体 | 疲劳、嗜睡、痘痘、腹胀、食欲变化 |
| 生活 | 睡眠不好、运动、饮酒、压力大 |
| 备注 | 自由文本，最多 200 字 |

疼痛可选 0-10 强度；其他症状以多选标签为主，减少输入成本。

### 4.5 周期预测与提醒

#### 4.5.1 预测规则
- 初始预测使用“最近一次经期开始日 + 用户填写的周期长度”计算。
- 当用户记录满 3 个完整周期后，使用最近 3-6 个周期的平均周期长度进行预测。
- 若周期波动较大，页面文案使用“预计”而不是“确定”，并显示“预测可能不准，请以实际身体情况为准”。
- 易孕期、排卵日和备孕建议不进入 MVP，避免过早引入医学和避孕风险。

#### 4.5.2 提醒规则
- 经期将至提醒：预计开始前 1-3 天提醒，可关闭。
- 每日记录提醒：用户可设置固定时间，通知文案必须脱敏，例如“今天记一下状态吗？”。
- 经期结束确认：经期预测结束日后提醒用户确认是否结束。
- 通知不得展示“经期”“排卵”“备孕”等敏感词，除非用户明确开启“详细通知”。

### 4.6 隐私与安全边界

妈妈工具的数据比孩子习惯数据更敏感，默认采用更严格的隐私策略：

- 妈妈工具默认不展示具体内容，进入页面后可要求私密解锁。
- MVP 可先使用应用内数字 PIN；后续接入系统生物识别。
- App 进入后台、锁屏或超过 5 分钟未操作后，妈妈工具自动重新锁定。
- 妈妈工具数据不出现在任务页、学习页、我的成长页、孩子勋章墙和家长数据看板。
- 默认本地保存；云同步必须单独开启，并说明同步范围、用途和删除方式。
- 用户可以一键清除妈妈工具数据；清除前需要二次确认。
- 不把经期、症状、备孕、同房、用药等数据用于广告、推荐、模型训练或儿童内容个性化。

### 4.7 非医疗边界

- 本功能只用于个人记录和生活提醒，不提供疾病诊断、治疗建议、避孕结论、妊娠判断或用药建议。
- 周期预测、经期提醒和未来可能加入的易孕期提示都不能作为避孕依据。
- 当用户记录“异常大量出血、剧烈疼痛、长期不规律、疑似怀孕”等情况时，产品只能提示“如有担心，请咨询医生”，不做自动诊断。
- 正式上线前，隐私政策需要将女性健康数据作为敏感个人信息单独说明。

### 4.8 后续可扩展能力

| 能力 | 价值 | 建议阶段 |
|------|------|----------|
| PDF / CSV 导出 | 方便复诊或长期回顾 | V2 |
| 基础体温记录 | 服务备孕用户 | V2 |
| 排卵试纸 / 宫颈黏液记录 | 更完整的备孕记录 | V2 |
| 避孕药 / 药物提醒 | 帮助管理固定用药 | V2，需更强隐私和免责声明 |
| Apple Health / 系统健康数据互通 | 减少重复记录 | V3，先评估平台能力和合规 |
| 多主题工具 | 体重、睡眠、喝水、心情日记 | V3 |

### 5.1 数学小岛（学习乐园子模块）

数学小岛重点服务幼升小数学准备：数感、数量关系、空间图形、规律推理和简单应用题。建议先把"能力点"作为底层数据，再把能力点包装成岛屿关卡。

> MVP 仅在学习 Tab 中保留数学小岛入口和“内容准备中”占位页，本章能力在后续学习阶段实施。

### 5.1.1 世界观设计
数学小岛由多个岛屿组成，每座岛屿包含 5 个关卡 + 1 个 Boss 关。孩子坐船从一个岛到下一个岛。

```
🏝️ 数数岛（1-10）  →  🍎 比大小岛  →  ➕ 加法岛（10以内）
        ↓
🔟 数数岛2（11-20）  →  🧩 形状岛  →  ➖ 减法岛（10以内）
        ↓
🔢 数数岛3（1-50）  →  📏 排序岛  →  ✏️ 混合运算岛（20以内）
```

### 5.1.2 幼升小数学能力地图

| 能力点 | 幼升小目标 | 推荐关卡形式 | 本地记录指标 |
|--------|-----------|-------------|-------------|
| 唱数与点数 | 能从 1 数到 50，点数时不漏不重 | 点物计数、接龙数数、缺失数字 | 最大稳定数域、点数正确率 |
| 数量对应 | 理解"几个"和物体一一对应 | 给动物分食物、按数量取物 | 一一对应正确率 |
| 数字识别 | 认识 0-20，逐步扩展到 50 | 数字泡泡、数字找朋友 | 数字识别正确率、易错数字 |
| 数序与相邻数 | 知道前一个/后一个，能补全数列 | 火车车厢排序、数字阶梯 | 相邻数正确率 |
| 比较大小 | 理解多/少/一样多，数字大小 | 左右两堆比较、拖拽 > < = | 比较题正确率 |
| 5/10 的分合 | 理解 5 和 10 的组成 | 拼积木、分糖果、十格阵 | 分合掌握度 |
| 10 以内加减 | 能用实物理解加减含义 | 小鸟飞来飞走、糖果增减 | 首次正确率、是否依赖实物 |
| 图形认知 | 认识圆形、三角形、正方形、长方形 | 找形状、形状拼图 | 图形识别正确率 |
| 空间方位 | 理解上下左右、里外前后 | 找位置、迷宫移动 | 方位词正确率 |
| 规律排序 | 能发现 AB、AAB、ABC 等简单规律 | 珠子串、颜色序列 | 规律补全正确率 |
| 分类统计 | 能按颜色、形状、大小分类 | 给物品归类、数一数哪类多 | 分类正确率 |
| 简单应用题 | 能听懂短句数学情境 | 绘本式题目、语音题目 | 题干理解正确率 |

### 5.1.3 数学岛屿推荐顺序

学习模块数学首版建议做 6 个核心岛屿，覆盖幼升小最常见能力：

1. 数数岛：1-10 点数、取物、一一对应。
2. 数字岛：0-20 数字识别、相邻数、数字排序。
3. 比较岛：多/少/一样多、大于/小于/等于。
4. 分合岛：5 的分合、10 的分合、凑十启蒙。
5. 加减岛：10 以内加法、10 以内减法。
6. 图形规律岛：基础图形、空间方位、简单规律。

20 以内加减和混合运算建议放在 V2，不建议 MVP 过早加重。

### 5.1.4 关卡结构
每个岛屿 = 5 个普通关 + 1 个 Boss 关
- 普通关：首次解锁消耗 1 颗小心心，解锁后永久有效
- Boss 关：需通关该岛全部 5 个普通关，不消耗小心心
- Boss 关有时间限制（倒计时 60 秒），答对 5 题即通过

### 5.1.5 关卡类型详设

#### 5.1.5.1 数数关（Counting）
```
屏幕显示：画面上散落着数量不等的小动物/水果/星星
操作方式：点一个，数一个（被点的物品放大 + 显示数字标记1,2,3...）
问题形式：
  - "这里有几只小兔子？"（自由数，底部有数字键盘）
  - "拿走 3 个苹果"（按数量点击选取）
  - "给每个小动物分 1 个胡萝卜"（一一对应拖拽）
答对反馈：数字变大飞向屏幕中央 + 星光特效（不增加星星余额）
答错反馈：物品闪烁提示重新数 + "再数一次试试？"
```

#### 5.1.5.2 比大小关（Comparison）
```
屏幕显示：左右两堆物品
操作方式：点击更大/更小的一边；或拖拽中间符号 > < =
问题形式：
  - "哪边的小鱼更多？"（点选）
  - "苹果比香蕉多还是少？"（选择"多"或"少"按钮）
  - "放上正确的符号"（拖拽 >  <  = 到两堆物品中间）
  - "找出两组一样多的"（视觉匹配）
答对反馈：大的一边高亮闪动
```

#### 5.1.5.3 加法/减法关（Addition / Subtraction）
```
屏幕显示：上半部分是物品图示，下半部分是算式
操作方式：拖拽物品 / 点击数字按钮
问题形式：
  - "树上有 3 只小鸟，又飞来 2 只，现在有几只？"（小鸟飞入动画）
  - "你有 5 个糖果，吃掉了 2 个，还剩几个？"（糖果消失动画）
  - "2 + 3 = ？"（下方数字键盘输入）
支持的辅助：
  - 手指点物品一个一个数
  - 物品可以拖到计数区辅助数数
答对反馈：算式变金色 + 物品集体跳舞
答错反馈：物品变灰等待 + "可以数一数哦~"提示
```

#### 5.1.5.4 形状与图案关（Shapes & Patterns）
```
形状识别：
  - "哪个是圆形？"（4 个形状中选 1）
  - "生活中哪些东西是圆形的？"（图片选择：钟表✓ / 书本✗ / 球✓）
  - 形状拼图：拖拽形状到正确位置完成图案

图案规律：
  - "下一个是什么？" 🔴🔵🔴🔵 ？ → 选 🔴
  - "缺了什么？" 给出不完整的图案矩阵，选缺失的那个
```

#### 5.1.5.5 Boss 关
```
形式：问答马拉松
题目数量：5 题
时间限制：60 秒倒计时（沙漏动画）
题型：该岛屿所有类型的混合
Boss 形象：与该岛主题相关的可爱怪物
  - 数数岛 Boss：爱吃数字的数字怪
  - 加法岛 Boss：拿着大算盘的算盘龙
通过条件：5 题全对
通过奖励：🏝️ 岛屿点亮 + 角色新装扮 + Boss 收藏卡
失败：可以重试，不消耗小心心
```

### 5.1.6 进度与激励
- 每个岛屿完成 → 地图上该岛屿点亮 + 获得 Boss 收藏卡
- 通关 3 个岛屿 → 解锁"数学小勇士"称号 + 剑装扮
- 通关全部岛屿 → 解锁"数学大师"称号 + 魔法棒装扮
- 所有题目 100% 正确率 → 获得金色通关标记

### 5.1.7 数学掌握度规则

为了让家长看得懂进度，建议每个能力点使用 4 档掌握状态：

| 状态 | 判定规则 | 展示文案 |
|------|---------|---------|
| 未开始 | 没有完成任何相关题目 | 还没开始探索 |
| 认识中 | 完成题目少于 10 道，或正确率 < 70% | 正在认识 |
| 基本掌握 | 最近 10 道正确率 ≥ 70% | 基本会啦 |
| 稳定掌握 | 最近 20 道正确率 ≥ 85%，且连续 3 次通过关卡 | 很稳了 |

Boss 关不只判断通关，还应写入能力点表现，帮助数据看板提示"哪个能力还需要陪练"。

### 5.1.8 固定知识与数学题库存储策略

小学低年级知识点相对稳定，但题目仍建议进入内容管理体系，不建议全部硬编码在 App 中。

| 内容 | 推荐存储位置 | 原因 |
|------|-------------|------|
| 数学能力点、岛屿、关卡 | CloudBase MySQL | 便于调整顺序、难度和上下架状态 |
| 少量图形题、情境题、Boss 题 | `math_questions` 固定题表 | 每道题的图片、选项和讲解需要人工审核 |
| 数数、比较、分合、10 以内加减 | `math_question_templates` 题目模板表 | 用参数规则生成多道题，避免重复录入大量相似题 |
| 题目图片、动画、音效 | CloudBase 云存储 | 数据库只保存资源键和版本，不直接保存大文件 |
| 学习模块基础题包 | App 内置 JSON + Drift 缓存 | 无网可用，首次打开不需要等待下载 |
| 答题与掌握度 | CloudBase MySQL + Drift 待同步队列 | 支持多设备同步、家长看板和弱网补传 |

**推荐做法：**
- CloudBase 是内容“源数据”，安装包内置的是某个已审核版本的快照。
- 每道生成题保存模板编号、生成参数和题目快照，保证之后仍能还原孩子当时答过的题。
- 不追求无限随机。幼升小题目应设置数值范围、干扰项规则和难度上限，并经过内容审核。
- 数学首版可准备约 80-120 道固定题，加 10-20 个题目模板。

### 5.2 英语森林（学习乐园子模块）

英语森林重点服务幼升小英语启蒙：兴趣、听辨、敢开口、字母认知、基础词汇和课堂常用表达。首版不依赖在线语音评测，先用已下载到本地的音频、点击选择、跟读自评和家长确认完成闭环。

> MVP 仅在学习 Tab 中保留英语森林入口和“内容准备中”占位页，本章能力在后续学习阶段实施。

### 5.2.1 世界观设计
英语森林是一棵巨大的魔法树，树上有 26 个树屋（对应 26 个字母）。每个树屋里住着一个字母精灵。随着学习推进，树从光秃秃变得枝繁叶茂。

```
🌳 魔法字母树
├── 🔤 字母认知区（26个字母，分5批解锁）
├── 📝 单词小镇（按主题：动物、食物、颜色、身体、家庭）
├── 🎵 儿歌谷（字母歌、颜色歌、数字歌）
├── 🗣️ 对话小屋（简单日常问候）
└── 📚 绘本小路（V2，自有或已获得数字版权的分级绘本）
```

### 5.2.2 幼升小英语能力地图

| 能力点 | 幼升小目标 | 推荐关卡形式 | 本地记录指标 |
|--------|-----------|-------------|-------------|
| 英语兴趣 | 愿意听、愿意点、愿意跟唱 | 儿歌、动画卡片、角色互动 | 主动学习天数、重复播放次数 |
| 听音辨识 | 听到音频能选出对应图片/字母 | 听音选图、听音选字母 | 听辨正确率 |
| 字母认知 | 认识 26 个大写字母，逐步认识小写 | 字母泡泡、树屋点亮 | 大写识别率、小写识别率 |
| 字母音启蒙 | 知道部分字母常见发音，不要求拼读完整 | A says /a/、首音配对 | 字母音正确率 |
| 基础词汇 | 掌握常见主题词 50-80 个 | 图片选词、听音选图、翻翻乐 | 主题词掌握数、易错词 |
| 日常表达 | 能听懂并模仿简单问候 | 角色对话、跟读按钮 | 跟读次数、家长确认 |

| 课堂指令 | 听懂 sit down / stand up / look / listen 等 | 指令小游戏 | 指令理解正确率 |
| 分级阅读 | 能结合图片理解重复句型和故事顺序 | 逐页听读、点词发音、图片排序 | 阅读完成数、复听次数、理解题正确率 |

### 5.2.3 英语内容推荐范围

英语首版不建议做太多单词和复杂拼读，先保持轻量、可完成：

| 模块 | 英语首版范围 | 后续范围 |
|------|---------|---------|
| 字母 | 26 个大写 + 常见小写配对 | 字母音、首音分类 |
| 单词 | 动物、食物、颜色 3 个主题，每主题 6-10 个 | 身体、家庭、物品、自然 |
| 儿歌 | 3-5 首短儿歌 | 8-12 首，增加跟唱模式 |
| 对话 | Hello / Bye / Thank you / I like... | 更多场景对话 |
| 发音 | 本地音频跟读 + 家长确认 | 设备端语音识别或在线能力预留 |
| 分级绘本 | 暂不内置商业绘本；可做 2-3 本自有短故事验证流程 | 接入自有、公共领域或正式授权的连续分级绘本 |

### 5.2.4 字母认知关卡

#### 5.2.4.1 关卡结构
字母分 5 批解锁，每批 5-6 个字母：
- 第 1 批：A B C D E
- 第 2 批：F G H I J
- 第 3 批：K L M N O
- 第 4 批：P Q R S T
- 第 5 批：U V W X Y Z

每批首次解锁消耗 2 颗小心心，解锁后永久有效。

#### 5.2.4.2 单字母学习流程（每个字母 3 个小环节）
```
环节一：认识字母（约 30 秒）
  - 屏幕中央大字母 "A"
  - 发音播放："A, A, /eɪ/, Apple 苹果"
  - 字母周围飞出 A 开头的物品动画（Apple 🍎, Ant 🐜, Alligator 🐊）
  - 孩子可反复点击字母听发音

环节二：找字母（游戏化）
  - 屏幕上散落 6-8 个字母泡泡（包含目标字母和干扰字母）
  - "找到所有的 A！"
  - 点击正确的字母泡泡 → 泡泡炸开 + 发音 + 星光特效（不增加星星余额）
  - 点击错误的字母 → 泡泡弹开（不扣分）+ "我是 B 哦~"
  - 找到所有目标字母 → 环节完成

环节三：字母描红（书写启蒙）
  - 大字母 A 的描红路径（虚线 + 箭头指示笔画方向）
  - 孩子手指沿着路径描一遍
  - 描完字母变彩色 + 字母精灵出现说"You did it!"
  - 描得不够好也不判定失败，鼓励为主
```

#### 5.2.4.3 字母复习关卡
每批字母学完 → 有一个复习关：
- **字母配对**：大小写配对（拖拽小写 a 到大写 A 旁边）
- **字母排序**：把散落的字母按字母表顺序排成一排
- **听音选字母**：播放发音 /eɪ/，在 4 个字母中选 "A"

### 5.2.5 单词小镇

#### 5.2.5.1 主题分类
| 主题 | 单词量 | 示例单词 |
|------|--------|---------|
| 🐱 动物 | 10 个 | cat, dog, fish, bird, rabbit, bear, pig, cow, duck, monkey |
| 🍎 食物 | 10 个 | apple, banana, milk, egg, bread, cake, water, grape, orange, rice |
| 🎨 颜色 | 8 个 | red, blue, green, yellow, orange, purple, pink, white |
| 👀 身体 | 8 个 | eye, ear, nose, mouth, hand, foot, head, arm |
| 👨‍👩‍👧 家庭 | 6 个 | mom, dad, baby, brother, sister, grandma |
| 🏠 物品 | 10 个 | book, ball, chair, table, bed, door, car, sun, star, tree |

#### 5.2.5.2 单词学习流程（每个单词）
```
环节一：看图识词
  - 大图片（如 🐱） + 下方单词 CAT
  - 播放发音 "Cat"
  - 孩子点击 → 猫咪叫一声 + 重复发音
  - 下方显示中文"猫"

环节二：图片选词
  - 屏幕上方：一张图片（如 🐶）
  - 屏幕下方：3 个单词按钮（DOG / CAT / BIRD）
  - 选对 → ✓ + 发音 + 狗狗摇尾巴动画
  - 选错 → 错误的单词变灰 + "再试试！"
  - 每个单词需连续答对 2 次才算通过

环节三：听音选图
  - 播放发音 "Banana"
  - 屏幕显示 4 张图片（🍌 🍎 🍇 🥛）
  - 点击正确的图片
  - 选对 → 香蕉蹦跳 + "Banana!"
```

#### 5.2.5.3 主题复习
每个主题学完 → 复习小游戏：
- **打地鼠**：地鼠从洞里钻出，举着单词牌，听音打正确的地鼠
- **翻翻乐**：记忆翻牌，翻开两张卡片，图片和单词配成一对
- **画画猜词**：屏幕上逐渐画出物品轮廓，孩子尽早猜出单词

### 5.2.6 儿歌谷
- 使用短句重复、动作明确、节奏舒缓的原创或合法授权儿歌
- 配有字幕 + 关键词高亮 + 简单动画
- 首版题材建议：原创字母韵律、颜色韵律、数字 1-10、身体动作、动物叫声
- 传统童谣也要确认具体歌词、编曲和录音的权利；即使旋律进入公共领域，现代编曲和第三方录音仍可能受保护
- 每首首次解锁消耗 1 颗小心心，解锁后永久有效
- 跟唱模式：每句播放后停顿留白，鼓励孩子跟唱
- 歌词中关键单词用图片代替文字（如 🌟 代替 star）

### 5.2.7 对话小屋（进阶，V2 功能）
- 简单情景对话："Hello! How are you?" / "I'm good, thank you!"
- 角色扮演：孩子选择角色，和森林里的动物对话
- 语音输入（可选，V2）：用语音识别判断孩子发音

### 5.2.8 进度与激励
- 每学完 1 个字母 → 魔法树上长出叶子
- 每学完 1 批字母 → 对应的树屋亮灯
- 每学完 1 个主题单词 → 单词小镇多一个建筑
- 集满整棵树 → 获得"英语魔法师"称号
- 连续学习 7 天 → 树开花特效

### 5.2.9 英语掌握度规则

英语模块建议避免用"考试分数"表达，使用孩子更友好的状态：

| 状态 | 判定规则 | 展示文案 |
|------|---------|---------|
| 新朋友 | 第一次接触字母/单词 | 刚认识 |
| 见过啦 | 完成学习环节，但复习正确率不足 70% | 有点熟 |
| 认得出 | 最近 10 次选择正确率 ≥ 70% | 能认出 |
| 听得懂 | 听音选择正确率 ≥ 80% | 听得懂 |
| 会开口 | 完成跟读，且家长确认愿意模仿 | 敢说啦 |

对于首版未接入语音识别的情况，"会开口"不自动判分，由家长在弹窗中轻量确认即可。

### 5.2.10 字母与单词描边、固定词汇发音

这两类能力适合在英语模块正式建设时实现：

#### 5.2.10.1 字母描边
- 为 A-Z、a-z 保存标准描边路径、起笔点、笔画顺序和容错宽度。
- 孩子手指移动时计算触点到标准路径的距离，超出容错区时只做柔和提示，不判定失败。
- 首版记录完成次数、是否独立完成和平均用时，不做书写质量打分。
- 描边路径可以随基础内容包下载，也可以随安装包内置，不需要逐次查询云端。

#### 5.2.10.2 单词描边
- 先覆盖 30-50 个高频短词，每次只描一个单词，建议 3-6 个字母。
- 可提供“看图 -> 听词 -> 描边 -> 再听一次”的闭环。
- 单词字体需要使用适合儿童识字的标准字形，避免花体和容易混淆的字母造型。

#### 5.2.10.3 固定词汇发音
- 优先使用获得授权的真人录音，按“单词慢速、单词正常速度、例句”拆成独立音频片段。
- 音频文件放 CloudBase 云存储，MySQL 只保存资源键、口音、说话人、时长、版本和授权编号。
- 已下载音频缓存到本地，可反复播放，不需要每次联网。
- Android 系统 TTS 只作为资源缺失时的降级方案，不作为正式教学发音源。

### 5.2.11 连续分级绘本与牛津树内容策略

产品可以借鉴牛津树的“连续角色、逐级增加词汇和句型、重复阅读、阅读理解”方法，但不能在未获得授权时复制牛津树的书名、角色、插画、正文、电子书文件或配套录音。

#### 5.2.11.1 产品实现方式
- 数据结构预留“系列 -> 级别 -> 图书 -> 页面 -> 文字/图片 -> 音频片段 -> 理解题”。
- 页面可支持整页朗读、逐句播放、点词发音、当前句高亮和阅读进度保存。
- 音频片段记录页面、句子起止时间；后续可以实现字幕跟随和逐句复听。
- 图书图片、音频文件放 CloudBase 云存储，数据库只保存结构、资源键和授权状态。
- 授权内容应使用受控下载地址，并支持到期下架、禁止导出和本地缓存清理。

#### 5.2.11.2 内容来源优先级
1. **首选：自有原创分级读物。** 自己设计角色和故事，聘请插画师、编辑和配音员，并取得完整数字发行授权。
2. **可选：公共领域故事再创作。** 只使用确认已进入公共领域的原始文本；新译文、插画和录音仍需自有或单独授权。
3. **商业合作：正式采购数字版权。** 与出版社确认中国大陆地区、App 内展示、音频、离线缓存、用户规模、授权期限和续费方式。
4. **不采用：从网页、网盘或已购买账号中抓取电子稿和音频。** 个人可阅读不等于可以复制并在自己的 App 中分发。

#### 5.2.11.3 牛津树接入判断
- Oxford Owl 和 Oxford Reading Buddy 提供官方在线电子书及音频相关能力，但公开页面展示的是面向家庭或学校的账号访问与订阅服务。
- 本次官方资料检查未发现面向第三方 App 的公开内容接口，因此不能按“直接抓取或嵌入”来设计；若要内置牛津树，应先联系 Oxford University Press 的 Education/Children's Rights 团队取得数字内容许可和文件交付方案。
- 在授权完成前，产品文案使用“原创分级绘本”或“连续分级阅读”，不要使用“牛津树课程”“官方牛津树”等可能造成授权或品牌关系误解的表述。

## 七、全局设计规范

### 7.1 视觉设计原则
- **色彩**：明亮、温暖、饱和度高（马卡龙色系为主，避免荧光色）
- **圆角**：所有卡片、按钮使用大圆角（16-24dp）
- **图标**：大图标（最小 48dp），避免小文字
- **字体**：中文用圆体/卡通体，英文用 Comic Sans 或类似圆润字体
- **字号**：孩子看的文字最小 18sp，按钮文字 22sp+
- **间距**：元素间距宽松，防止误触

### 7.2 动效原则
- 所有状态变化都要有过渡动画（不能突然闪现）
- 正面反馈动画要夸张（放大的 ✓、飞入的星星、蹦跳的角色）
- 负面反馈要温柔（轻微抖动、淡化，不能有❌或刺眼红色）
- 动画时长：快速反馈 200-300ms，庆祝动画 800-1500ms

### 7.3 音效原则
- 所有可交互元素都要有声音反馈
- 正反馈音效：清脆、上扬、明亮
- 负反馈音效：柔和、低沉、简短
- 背景音乐：轻快、柔和、音量可调
- 提供"静音模式"开关（家长可开启）

### 7.4 内容安全原则
- **无第三方广告**
- **无内购**（所有内容通过使用 App 解锁，不通过付费）
- **无社交功能**（不提供用户之间的数据共享、排行或互动）
- **云端存储**：家长账号、孩子档案、学习记录、习惯记录等数据存储在 CloudBase；妈妈健康数据默认本地保存，开启云同步前必须单独授权
- **内容版权**：只有自有、公共领域或取得书面数字授权的图片、文字、音乐、绘本和录音可以进入内容包
- **最小化采集**：不采集孩子真实姓名、身份证、学校、精确位置等非必要信息
- **监护人同意**：注册时明确由家长同意隐私政策与儿童个人信息保护规则
- **无外部链接**：防止孩子误触跳转

### 8.5 CloudBase 数据权限原则

- CloudBase Auth 内置用户系统维护手机号格式用户名、密码和登录态，业务数据库不保存明文密码。
- MVP 中手机号仅作为账号格式标识，不能视为已验证联系方式；正式发布前必须增加手机号归属验证。
- 用户私有表必须包含 `_openid VARCHAR(64) DEFAULT '' NOT NULL`，用于 CloudBase 用户级权限控制。
- 孩子档案、习惯、打卡、学习记录、奖励兑换等数据只能被所属家长账号读写。
- 妈妈工具数据必须与孩子数据分表隔离；默认不上传，开启云同步时需要独立授权、独立删除和通知脱敏。
- 题库、能力点、勋章模板、内容资源等模板表由后台维护，客户端只读。
- 移动客户端不保存 API Key；涉及管理权限的操作必须通过云函数或服务端接口完成。
- MVP 对重置孩子数据、删除孩子档案和导出数据使用家长区隔离 + 危险操作确认；正式上线前应升级为账号密码、短信或其他可靠二次验证。

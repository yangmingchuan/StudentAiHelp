# 闯关小勇士 MVP 基础版本需求

> 状态：已确认，进入技术设计  
> 版本：v0.2  
> 日期：2026-06-22  
> 上游文档：[PRD.md](../../PRD.md)

## 1. 问题与目标

当前项目已有完整产品蓝图，但首个版本同时覆盖任务、数学、英语、绘本和内容包，范围过大。MVP 需要先验证“家长创建任务，孩子每天打卡并获得成长反馈”这一核心闭环。

MVP 目标：

- 家长能够注册、登录并初始化一个孩子档案。
- 家长能够配置孩子的日常任务。
- 孩子能够完成、跳过或撤销当天任务。
- 系统能够正确结算星星、全完成奖励、连续奖励和任务勋章。
- 孩子默认不会误入家长设置区域。
- 数据能够保存到 CloudBase，并在弱网时保留本地可用性。

## 2. MVP 范围

### 2.1 包含

- 手机号格式账号 + 密码注册登录。
- 默认孩子档案初始化。
- 四 Tab 主框架。
- 今日任务、任务管理和打卡记录。
- 星星累计数与当前可用余额。
- 每日固定 10 颗小心心的展示与跨天补充。
- 任务成就勋章。
- 连续点击 5 次进入家长区。
- Drift / SQLite 本地缓存和待同步队列。
- CloudBase Auth、MySQL 和业务接口。

### 2.2 不包含

- 短信验证码、手机号归属验证和自助找回密码。
- 可玩的数学关卡和英语课程。
- 数学题库、英语词汇音频、字母描边和内容包下载。
- 分级绘本及商业版权内容。
- 星星礼品兑换的完整流程。
- 家长数据看板和角色装扮。
- 家长密码、短信二次验证或其他强家长认证。
- 多设备同步冲突自动合并。

## 3. 冻结业务规则

| 对象 | 规则 |
|------|------|
| 星星 | 任务完成后获得；记录累计获得数和当前可用余额；兑换只扣当前余额 |
| 奖励币 | MVP 不设置独立奖励币，星星承担未来礼品兑换货币作用 |
| 小心心 | 每天 0 点补满至 10 颗，不累加、不结转；MVP 没有实际消费场景 |
| 勋章 | 只展示任务成长成就，不消费、不兑换 |
| 普通任务奖励 | 每完成 1 个任务获得 1 颗星星 |
| 全完成奖励 | 当天全部任务完成额外获得 2 颗星星 |
| 连续奖励 | 连续 7 天全部完成额外获得 5 颗星星 |
| 历史修改 | 只允许修改当天打卡，跨天后历史记录只读 |
| 家长入口 | “我的”页面头部区域 3 秒内连续点击 5 次 |

## 4. 用户故事

1. 作为家长，我希望使用容易记住的账号注册并登录，以便保存孩子数据。
2. 作为家长，我希望快速设置和调整任务，以便任务符合孩子当前情况。
3. 作为孩子，我希望通过简单按钮记录任务结果，并立即看到星星反馈。
4. 作为孩子，我希望看到自己的任务勋章，以便感受到持续完成任务的成就。
5. 作为家长，我希望孩子不容易误改设置，以便日常使用更稳定。
6. 作为家庭用户，我希望弱网时仍能看到任务并打卡，以便网络波动不打断使用。

## 5. 功能需求与验收标准

### Requirement 1：账号注册与登录

- 当家长提交合法的 11 位手机号格式账号、合规密码并同意隐私条款时，系统应使用 CloudBase 用户名/密码能力创建账号。
- 当账号注册成功时，系统应将其标记为“手机号归属未验证”，且不得将该账号标识当作可信联系方式。
- 当已注册的手机号格式账号再次注册时，系统应拒绝创建重复账号并提示直接登录。
- 当家长输入正确账号和密码时，系统应创建登录会话并安全保存 AccessToken 与 RefreshToken。
- 当家长点击“登录遇到问题”时，MVP 应提示联系内测管理员，不提供短信找回密码。
- 当产品准备正式对外发布时，发布检查应要求接入手机号归属验证和可信密码找回流程。

### Requirement 2：孩子档案初始化

- 当家长首次注册成功时，系统应为该账号创建一个默认孩子档案。
- 当默认孩子档案缺少昵称时，系统应引导家长完成昵称、年龄段和头像初始化。
- 当家长账号读取孩子数据时，系统应只返回归属于当前账号的数据。

### Requirement 3：任务管理

- 当家长进入家长区时，系统应允许添加、编辑、删除和拖拽排序任务。
- 当任务数量少于 1 个或超过 10 个时，系统应阻止保存。
- 当任务名称超过 7 个汉字的等效长度时，系统应阻止保存并显示明确提示。
- 当任务被删除时，系统应保留历史打卡记录，不应物理删除孩子成长历史。

### Requirement 4：当天打卡

- 当孩子将未操作任务标记为完成时，系统应更新当天记录并立即展示完成反馈。
- 当孩子将未操作任务标记为跳过时，系统应记录跳过，且不增加或扣除星星。
- 当孩子在当天撤销完成状态时，系统应撤回该任务产生的星星，并恢复未操作状态。
- 当孩子尝试修改历史日期的任务记录时，系统应拒绝修改并保持记录只读。
- 当同一任务在同一天被重复提交相同状态时，系统应保持结果幂等，不得重复结算。

### Requirement 5：星星结算

- 当任务首次变为完成状态时，系统应为当前可用星星和累计获得星星各增加 1。
- 当当天所有有效任务均完成时，系统应额外增加 2 颗星星。
- 当撤销操作导致当天不再全部完成时，系统应撤回当天的 2 颗全完成奖励。
- 当重新满足当天全部完成条件时，系统应恢复全完成奖励，但同一奖励不得重复累计。
- 当当前日与此前连续 6 天均为全部完成时，系统应额外增加 5 颗连续奖励星星。
- 当打卡、全完成奖励或连续奖励请求因弱网被重复发送时，服务端应根据业务幂等键只结算一次。
- 当未来发生礼品兑换时，系统应只扣减当前可用星星，不扣减累计获得星星。

### Requirement 6：小心心

- 当孩子在一个新自然日首次打开 App 或请求今日数据时，系统应按服务端家庭时区将小心心补充至 10 颗。
- 当上一日仍有剩余小心心进入新自然日时，系统应将数量设为 10，而不是累加。
- 当设备时间被手动修改时，系统应以服务端日期判断是否跨天。
- 在 MVP 中，数学和英语内容不可玩时，系统不得产生小心心扣减记录。

### Requirement 7：任务勋章

- 当孩子满足勋章模板条件时，系统应发放对应任务勋章。
- 当同一勋章条件被重复计算时，系统应只保留一条已获得记录。
- 当孩子查看勋章墙时，系统应展示已获得状态、获得日期、未获得条件和当前进度。
- 在任何兑换或学习解锁流程中，系统不得扣减勋章。

### Requirement 8：家长区

- 当用户在“我的”页面头部区域 3 秒内连续点击 5 次时，系统应进入家长区并展示家长功能。
- 当点击次数未达到 5 次或超过时间窗口时，系统应重置点击计数。
- 当家长点击“退出家长区”时，系统应立即隐藏家长功能。
- 当 App 进入后台超过 5 分钟或重新启动时，系统应自动退出家长区。
- 当执行重置或删除类危险操作时，系统应再次显示明确确认弹窗。
- 系统不得把连续点击入口描述为安全身份认证；它只用于儿童误触隔离。

### Requirement 9：四 Tab 与学习占位

- 当用户登录成功时，系统应展示任务、数学、英语和我的四个 Tab。
- 当用户进入数学 Tab 时，MVP 应展示“内容准备中”占位页，不加载题库或产生小心心消耗。
- 当用户进入英语 Tab 时，MVP 应展示“内容准备中”占位页，不加载音频、描边、单词或绘本内容。
- 当用户重新打开 App 时，系统应默认进入任务 Tab。

### Requirement 10：本地缓存与同步

- 当网络不可用时，系统应展示最近一次成功同步的孩子档案、任务和当天状态。
- 当离线发生打卡操作时，系统应将操作写入带唯一操作 ID 的待同步队列。
- 当网络恢复时，系统应按创建顺序同步待处理操作，并避免重复发放星星或勋章。
- 当服务端结果与本地乐观状态不同，以服务端结算结果为准，客户端应更新显示。

## 6. 约束

- 客户端使用 Flutter + Dart，共享 Android/iOS 的 UI、业务、网络和本地数据代码。
- Android 最低支持 Android 8.0（API 26），iOS 最低支持 iOS 13。
- Android/iOS 允许保留少量平台工程代码，用于签名、权限、启动图和原生能力接入。
- CloudBase Auth 使用用户名/密码能力，用户名采用手机号格式。
- 移动客户端不得保存 CloudBase API Key。
- 星星、奖励和勋章结算必须通过可信服务端逻辑完成。
- 涉及儿童的数据遵循最小化采集原则。

## 7. 上线门槛

以下事项不阻塞 MVP 内测，但阻塞正式公开发布：

- 手机号短信验证或其他可信归属验证。
- 可用的密码找回流程。
- 比连续点击更可靠的家长认证。
- 儿童个人信息保护规则、隐私政策及监护人同意流程的法务确认。
- 删除账号、导出数据和清除儿童数据的正式流程。
